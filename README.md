## How I build

I start with the work itself:

1. understand the users and operating problem;
2. define the lifecycle, roles, rules, and edge cases;
3. build the smallest dependable workflow;
4. test real scenarios;
5. document decisions and release boundaries;
6. turn the result into clear public proof.

Development may use AI assistance. Product vision, requirements, workflow design, QA direction, validation, documentation, and final decisions remain under my ownership.
